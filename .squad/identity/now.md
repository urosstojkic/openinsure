# What We're Focused On

Building OpenInsure into a credible, enterprise-grade, AI-native insurance platform where Microsoft Foundry agents are the primary execution engine — not decoration.

## Current Sprint Goals

1. **Dashboard shows real SQL data with Foundry AI workflow** — Process buttons trigger real Foundry agents, show AI reasoning with confidence scores
2. **Every AI judgment goes through Foundry** — triage, underwriting, claims assessment, compliance audit
3. **Data is persistent and interconnected** — 200 submissions, 72 policies, 24 claims in Azure SQL
4. **Role-based views work end-to-end** — each persona sees their authorized data and can take actions within their authority

## Vision (from architecture spec)

"AI-native does not mean 'uses AI.' It means AI is the primary interface and execution engine, not an enhancement to a human-operated system. The default path is autonomous, with human oversight on exceptions."
